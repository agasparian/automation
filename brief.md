## ⚠️ ИЗМЕНЕНИЕ АРХИТЕКТУРЫ: нет Google Workspace

**Проблема:** без корпоративного домена Google Forms нельзя ограничить по домену.

**Решение: email-allowlist в Google Sheets + проверка в n8n**

```
[Google Form — открытая, но со сбором email]
        │
        ▼ новая строка
[n8n]
  └──► Проверить: email отправителя есть в листе «Допущенные»?
          │
          ├── НЕТ → отправить email «У вас нет доступа» → STOP
          │
          └── ДА → продолжить сценарий
```

**Лист «Допущенные» в Google Sheets:**
| Email | Имя | Роль |
|---|---|---|
| ivanov@gmail.com | Иванов А. | Менеджер |
| petrova@gmail.com | Петрова С. | Аккаунт |

Добавить нового сотрудника = добавить одну строку в таблицу. Удалить = удалить строку.
В форме обязательно включить «Собирать адреса электронной почты».

## КАКОЙ GMAIL-АККАУНТ ПОДКЛЮЧАТЬ К n8n

n8n подключается к Google через OAuth2. Нужно выбрать один Gmail-аккаунт, от имени которого n8n будет работать с Drive, Docs, Sheets.

**Рекомендация:** создать отдельный аккаунт — например `automation@gmail.com` или `n8n.medsprint@gmail.com`. Это изолирует автоматизацию от личных аккаунтов сотрудников.

**Что нужно сделать:**
1. Создать (или выбрать существующий) Gmail-аккаунт для n8n
2. Дать этому аккаунту доступ к:
   - Папке реестра на Drive (Editor)
   - Пилотному документу приложения (Editor)
   - Папке шаблонов (Editor)
   - Папкам клиентов (Editor)
3. Сообщить адрес этого аккаунта — чтобы настроить OAuth2 в n8n

## ФАЗА 1 — Google Form (триггер)

**Исполнитель:** администратор Google Workspace
**Зависит от:** Q1, завершения фазы 0.1

### Шаги:
1. Создать новую форму в Google Forms
2. Добавить поля:
   - **Клиент** — тип «Раскрывающийся список» (значения берутся из листа «Клиенты», обновлять вручную при добавлении клиента)
   - **Услуга** — тип «Раскрывающийся список» (значения из листа «Шаблоны»)
   - **Дата приложения** — тип «Дата», обязательное
   - **Дата начала работ** — тип «Дата», обязательное
   - **Срок действия** — тип «Дата», необязательное
   - **Стоимость (₽/мес)** — тип «Число», необязательное
   - **Дополнительные условия** — тип «Текст абзаца», необязательное
3. Настройки формы → «Только для пользователей домена [вашдомен.ru]»
4. Включить «Собирать адреса электронной почты» — чтобы знать, кто отправил запрос
5. Привязать форму к Google Sheets → создастся лист «Ответы на форму»

**Результат фазы 1:** форма работает, ответы пишутся в Sheets, доступна только сотрудникам.

## ФАЗА 3 — n8n: логика получения данных клиента

**Исполнитель:** разработчик
**Зависит от:** фазы 2, завершения фазы 0.1

### Шаги:
1. *(только при Google Forms без Workspace)* Добавить ноду **Google Sheets** (Read):
   - Лист: «Допущенные»
   - Фильтр: `Email == {{form.email}}`
   - Ноду **IF**: email найден? Нет → **Send Email** «У вас нет доступа» → **STOP**

2. Добавить ноду **Google Sheets** (Read):
   - Лист: «Клиенты»
   - Фильтр: `Название клиента == {{form.Клиент}}`
   - Результат: ID папки, ID файла реквизитов, последний номер приложения

3. Добавить ноду **IF** — клиент найден?
   - **Нет** → **Send Email** «Клиент не найден, проверьте написание» → **STOP**
   - **Да** → продолжить

4. Добавить ноду **Google Docs** (Get Document):
   - Document ID: `{{клиент.ID_файла_реквизитов}}`
   - Результат: текст документа с реквизитами

5. Добавить ноду **Code** (JS) или **Claude API** — извлечь поля из текста реквизитов:
   ```js
   // Парсинг строк формата «Ключ: Значение»
   const lines = text.split('\n');
   const fields = {};
   lines.forEach(line => {
     const [key, ...val] = line.split(':');
     if (key && val) fields[key.trim()] = val.join(':').trim();
   });
   return fields; // { «Полное название»: ..., «ИНН»: ..., ... }
   ```

6. Добавить ноду **Set** — вычислить новый номер:
   ```
   новый_номер = последний_номер + 1
   ```

7. Добавить ноду **Google Sheets** (Read):
   - Лист: «Шаблоны»
   - Фильтр: `Услуга == {{form.Услуга}}`
   - Результат: ID шаблона на Google Drive

8. Добавить ноду **IF** — шаблон найден?
   - **Нет** → **Send Email** «Шаблон для услуги [X] не найден» → **STOP**
   - **Да** → продолжить

**Результат фазы 3:** n8n знает номер нового приложения, ID шаблона, все реквизиты клиента.

## ФАЗА 4 — n8n: создание документа

**Исполнитель:** разработчик
**Зависит от:** фазы 3, завершения фазы 0.2

### Шаги:
1. Добавить ноду **Google Drive** (Copy File):
   - Source File ID: `{{шаблон.ID}}`
   - Destination Folder ID: `{{клиент.ID_папки}}`
   - Новое имя: `Приложение №{{новый_номер}} — {{клиент.Название}}`
   - Результат: ID нового документа

2. Добавить ноду **Google Docs** (Get Document) — получить тело документа для проверки (опционально)

3. Добавить несколько нод **Google Docs** (Replace Text) — по одной на каждый плейсхолдер:
   ```
   {{НОМЕР_ПРИЛОЖЕНИЯ}}       → новый_номер
   {{ДАТА_ПРИЛОЖЕНИЯ}}        → form.Дата_приложения (форматировать: «16 марта 2026 г.»)
   {{НАЗВАНИЕ_КЛИЕНТА}}       → клиент.Название
   {{ИНН_КЛИЕНТА}}            → клиент.ИНН
   {{АДРЕС_КЛИЕНТА}}          → клиент.Адрес
   {{ФИО_РУКОВОДИТЕЛЯ}}       → клиент.ФИО
   {{ДОЛЖНОСТЬ_РУКОВОДИТЕЛЯ}} → клиент.Должность
   {{УСЛУГА}}                 → form.Услуга
   {{ДАТА_НАЧАЛА_РАБОТ}}      → form.Дата_начала_работ
   {{СРОК_ДЕЙСТВИЯ}}          → form.Срок_действия (если пусто → «бессрочно»)
   {{СТОИМОСТЬ}}              → form.Стоимость (если пусто → из реестра последнего приложения)
   ```

   > 💡 Альтернатива множеству нод: одна нода HTTP Request к Google Docs API с batchUpdate — заменяет всё за один вызов. Рекомендую для чистоты сценария.

**Результат фазы 4:** готовый документ лежит в папке клиента на Google Drive.

## ФАЗА 5 — n8n: обновление реестра и уведомление

**Исполнитель:** разработчик
**Зависит от:** фазы 4

### Шаги:
1. Добавить ноду **Google Sheets** (Append Row):
   - Лист: «Реестр приложений»
   - Данные: дата, клиент, услуга, новый номер, ссылка на Doc (`https://docs.google.com/document/d/{{doc.ID}}`), статус «Черновик», email отправителя формы

2. Добавить ноду **Google Sheets** (Update Row):
   - Лист: «Клиенты»
   - Найти строку клиента → обновить «Последнее приложение №» и «ID последнего Doc»

3. Добавить ноду **Send Email** (или Telegram):
   ```
   Кому:   {{form.email_отправителя}}
   Тема:   Готово: Приложение №{{новый_номер}} — {{клиент.Название}}
   Тело:
   Приложение создано автоматически. Проверьте и отправьте клиенту.

   Документ: https://docs.google.com/document/d/{{doc.ID}}
   Клиент:   {{клиент.Название}}
   Услуга:   {{form.Услуга}}
   Дата:     {{form.Дата_приложения}}
   ```

**Результат фазы 5:** реестр обновлён, менеджер получил письмо со ссылкой.

## ФАЗА 6 — Тестирование

**Исполнитель:** разработчик + сотрудник по документообороту

### Тест-кейсы:

| # | Сценарий | Ожидаемый результат |
|---|---|---|
| T1 | Штатный случай: клиент есть, шаблон есть, все поля заполнены | Документ создан, реестр обновлён, email отправлен |

| T6 | Два запроса подряд для одного клиента | Документы с №4 и №5, оба в правильной папке |

### Критерий готовности:
Все 6 тест-кейсов проходят без ручного вмешательства.
