# Процесс: Создание нового приложения к договору

> Статус: готов к автоматизации
> Владелец процесса: сотрудник по документообороту
> Частота: по запросу (предположительно 5–15 раз в месяц)

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (триггер)

**Текущий триггер (ручной):**
Сотрудник получает запрос на новое приложение к договору — устно, в мессенджере или по почте. Кто-то (менеджер по продажам, аккаунт-менеджер) говорит: «Нужно приложение для клиента X на услугу Y».

**Данные, которые нужны на входе:**
| Поле | Пример | Обязательно? |
|---|---|---|
| Название клиента | ООО «Здоровье Плюс» | ✅ |
| Услуга (тип приложения) | Поддержка сайта | ✅ |
| Дата приложения | 16.03.2026 | ✅ |
| Дата начала работ | 01.04.2026 | ✅ |
| Срок действия / окончания | 01.04.2027 | зависит от услуги |
| Стоимость | 30 000 ₽/мес | зависит от услуги |
| Дополнительные условия | — | нет |

**Проблема:** нет единого места, куда поступает запрос. Информация разбросана по чатам.

---

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Шаг 1 — Найти последнее приложение клиента
**Кто:** сотрудник по документообороту
**Что делает:**
- Открывает Google Sheets (реестр договоров и приложений)
- Находит строку с нужным клиентом
- Смотрит колонку с последним приложением → находит ссылку на Google Doc
- Запоминает / записывает текущий номер приложения (например, «Приложение №3»)

**Время:** 3–7 минут
**Данные нужны:** название клиента, доступ к Google Sheets реестру

---

### Шаг 2 — Создать копию приложения в той же папке
**Кто:** сотрудник по документообороту
**Что делает:**
- Открывает найденный Google Doc
- Нажимает «Файл → Создать копию»
- Указывает название новой копии: меняет номер приложения (+1 к предыдущему)
  - Было: `Приложение №3 — ООО Здоровье Плюс`
  - Стало: `Приложение №4 — ООО Здоровье Плюс`
- Сохраняет копию в **ту же папку на Google Drive**, что и оригинал

**Время:** 3–5 минут
**Данные нужны:** текущий номер приложения, ID папки Google Drive клиента

---

### Шаг 3 — Взять контент из эталонного приложения (по типу услуги)
**Кто:** сотрудник по документообороту
**Что делает:**
- Открывает приложение другого клиента, которое является «образцом» для данной услуги
  (например: последнее подписанное приложение на «Поддержку сайта» от любого другого клиента)
- Выделяет и копирует **весь контент**, кроме блоков с подписями (реквизиты сторон, подписи, печати)
- Вставляет скопированный контент в новый документ (созданный на шаге 2), заменяя старое содержимое
- Блоки с подписями **оставляет от клиента** (из шага 2) — они уже есть в документе

**Время:** 5–10 минут
**Данные нужны:** понимание какой документ является «образцом» для данной услуги — нет явного реестра

> ⚠️ **Узкое место:** выбор «образца» зависит от памяти сотрудника. Нет формализованного списка шаблонов по услугам.

---

### Шаг 4 — Обновить переменные данные
**Кто:** сотрудник по документообороту
**Что делает:**
- Вручную находит и заменяет в документе:
  - Номер приложения (в заголовке и теле документа)
  - Дату приложения
  - Дату начала работ
  - Сроки (если есть)
  - Стоимость (если изменилась)
  - Другие специфические поля
- Проверяет, что название клиента в теле документа соответствует нужному клиенту

**Время:** 5–15 минут (зависит от количества переменных полей)
**Данные нужны:** все переменные из таблицы входных данных (см. раздел 1)

---

### Шаг 5 — Добавить запись в реестр Google Sheets
**Кто:** сотрудник по документообороту
**Что делает:**
- Возвращается в Google Sheets реестр
- Добавляет новую строку (или обновляет существующую) с данными нового приложения:
  ссылка на документ, дата, тип услуги, номер приложения, статус «на подписи»

**Время:** 3–5 минут

---

### Шаг 6 — Уведомить менеджера
**Кто:** сотрудник по документообороту
**Что делает:**
- Отправляет ссылку на готовый документ в чат менеджеру
- Менеджер проверяет, отправляет клиенту на подпись

**Время:** 1–2 минуты

---

### ИТОГО: 20–45 минут на одно приложение (при отсутствии ошибок)

---

## 3. ЧТО НА ВЫХОДЕ?

1. **Новый Google Doc** — приложение к договору с:
   - Правильным номером (последний + 1)
   - Актуальным содержанием (из эталона для данной услуги)
   - Обновлёнными переменными (дата, сроки, стоимость)
   - Блоком подписей данного клиента (реквизиты, ФИО)
   - Сохранён в папке клиента на Google Drive

2. **Обновлённая строка в Google Sheets** — реестр приложений пополнен

3. **Ссылка менеджеру** — документ готов к проверке и отправке клиенту

---

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ? (ошибки и исключения)

| # | Проблема | Последствие | Частота |
|---|---|---|---|
| 1 | Сотрудник ошибся в номере приложения (взял не последнее) | Дублирование номеров, путаница в реестре | Редко, но бывает |
| 2 | Копия сохранена не в ту папку на Drive | Документ «потерян», клиент не найдёт | Иногда |
| 3 | Взят не тот «образец» услуги | Неверный контент в приложении, переделка | Часто при новых сотрудниках |
| 4 | Не все переменные заменены | В документе осталось имя другого клиента или старая дата | Часто |
| 5 | Блок подписей случайно удалён/изменён | Нужно заново брать реквизиты клиента | Редко |
| 6 | Запрос пришёл с неполными данными | Сотрудник идёт уточнять, процесс встаёт | Регулярно |
| 7 | Клиент новый — нет предыдущих приложений | Нет от чего копировать номер/папку | Редко, но требует ручного старта |
| 8 | Реестр в Google Sheets не обновлён | Следующее приложение получит неверный номер | Регулярно |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Архитектура решения

```
[Менеджер заполняет Google Form]
        │  (только с корпоративным Google-аккаунтом)
        ▼
[Google Sheets — лист ответов формы (триггер)]
        │
        ▼ новая строка → webhook
[n8n — оркестратор]
        │
        ├──► [Google Sheets API] — читает реестр, получает последний номер + папку клиента
        │
        ├──► [Google Drive API] — создаёт копию эталонного шаблона в папке клиента
        │
        ├──► [Google Docs API] — заменяет плейсхолдеры реальными данными
        │
        ├──► [Google Sheets API] — добавляет строку в реестр приложений
        │
        └──► [Email / Telegram] — отправляет менеджеру ссылку на готовый документ
```

> **Почему Google Forms безопаснее Telegram Bot:**
> Форму можно настроить так, что заполнить её смогут только пользователи из вашего Google Workspace (`@вашдомен.ru`). Сторонний человек без корпоративного аккаунта не увидит форму вообще.

---

### Компоненты и инструменты

| Компонент | Инструмент | Что делает |
|---|---|---|
| Триггер | Google Forms | Менеджер заполняет форму — доступ только по корпоративному Google-аккаунту |
| Хранилище ответов | Google Sheets (авто) | Каждый ответ формы пишется в таблицу — отсюда n8n берёт данные |
| Оркестратор | n8n | Связывает все шаги в один сценарий |
| Реестр клиентов | Google Sheets API | Читает/пишет данные клиентов и приложений |
| Копирование файла | Google Drive API | Копирует шаблон в нужную папку, переименовывает |
| Замена переменных | Google Docs API | Find & Replace плейсхолдеров в документе |
| ИИ (опционально) | Claude API | Проверяет документ на ошибки, форматирует сложные поля |
| Уведомление | Email или Telegram | Отправляет менеджеру ссылку на готовый документ |

### Что нужно подготовить (предусловия)

#### A. Стандартизировать шаблоны
Создать отдельную папку на Google Drive: `Шаблоны приложений/`
- `Шаблон — Поддержка сайта.gdoc`
- `Шаблон — SEO-продвижение.gdoc`
- `Шаблон — Контекстная реклама.gdoc`
- ... (по каждой услуге)

Переменные поля в шаблонах размечать плейсхолдерами:
```
{{НОМЕР_ПРИЛОЖЕНИЯ}}
{{ДАТА_ПРИЛОЖЕНИЯ}}
{{НАЗВАНИЕ_КЛИЕНТА}}
{{ДАТА_НАЧАЛА_РАБОТ}}
{{СРОК_ДЕЙСТВИЯ}}
{{СТОИМОСТЬ}}
{{РЕКВИЗИТЫ_КЛИЕНТА}}   ← берётся из карточки клиента в Google Sheets
```

#### B. Настроить реестр в Google Sheets
Обязательные колонки:
| Клиент | ID папки Drive | Последнее приложение № | Ссылка на последний Doc | Дата последнего |
|---|---|---|---|---|

#### C. Карточка клиента в Google Sheets
Хранить реквизиты (ИНН, название, юр. адрес, ФИО руководителя) — для автозаполнения блока подписей.

### Сценарий работы после автоматизации

```
1. Менеджер открывает Google Form (ссылка в закладках браузера)
   → форма требует входа через корпоративный Google-аккаунт

2. Менеджер заполняет поля формы:
   - Клиент         → выпадающий список (подтягивается из реестра)
   - Услуга         → выпадающий список: SEO / Поддержка / Директ / ...
   - Дата приложения → поле даты (по умолчанию: сегодня)
   - Дата начала работ → поле даты
   - Стоимость      → числовое поле (необязательно — если не заполнено,
                       берётся из предыдущего приложения клиента)
   → нажимает «Отправить»

3. Google Sheets получает новую строку → n8n запускается автоматически:
   - Читает реестр → номер последнего приложения клиента = 3 → новый = 4
   - Копирует шаблон нужной услуги → в папку клиента на Drive
   - Переименовывает: «Приложение №4 — [Клиент]»
   - Заменяет все {{плейсхолдеры}} данными из формы + карточки клиента
   - Обновляет строку реестра (номер, ссылка, дата, статус «черновик»)
   - Отправляет менеджеру email со ссылкой на документ

4. Менеджер открывает документ, проверяет (30 секунд) → отправляет клиенту
```

**Время после автоматизации: 2–3 минуты** (только проверка менеджером)
**Экономия: 18–42 минуты на каждое приложение**

---

### Оценка сложности внедрения

| Этап | Сложность | Кто делает |
|---|---|---|
| Создать шаблоны с плейсхолдерами | Низкая | Сотрудник по документообороту |
| Настроить реестр в Google Sheets | Низкая | Сотрудник / менеджер |
| Создать Google Form + настроить доступ только для домена | Низкая | Администратор Google Workspace |
| Настроить n8n-сценарий (Forms → Drive + Docs + Sheets) | Средняя | Разработчик |
| Тест и отладка на 3–5 примерах | Низкая | Совместно |

**Итого на внедрение: 1–2 рабочих дня**

> **Настройка доступа к форме:** в Google Forms → Настройки → Ответы → включить «Только пользователи из домена [вашдомен.ru]». Форма становится невидима для всех, у кого нет корпоративного аккаунта.

---

### Следующий шаг

Чтобы начать автоматизацию, нужно:
1. Выбрать **одну услугу-пилот** (например, «Поддержка сайта»)
2. Создать шаблон Google Doc с плейсхолдерами
3. Проверить структуру реестра в Google Sheets
4. Написать мне — начнём строить n8n-сценарий
